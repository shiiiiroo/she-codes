
🚀 TaskFlow: Armanda AI
Интеллектуальный менеджер задач с ИИ-помощником, который помогает организовывать расписание через чат и голосовые команды.

🤖 Используемые технологии ИИ
В проекте задействованы две ключевые технологии ИИ:

Armanda AI (LLM): Использует модели OpenAI (GPT-4o/o1) или Claude (Anthropic) для обработки текстовых запросов, планирования задач и управления календарем. Она способна понимать команды вроде «перенеси рейд на завтра» или «создай тренировку на вечер».

Faster-Whisper: Локальная высокопроизводительная модель для транскрибации речи. Она превращает ваши голосовые сообщения в текст, который затем обрабатывается Armanda AI.

🛠️ Установка и запуск
1. Бэкенд (Python / FastAPI)
Перед запуском убедитесь, что вы находитесь в папке backend.

Создание и активация виртуального окружения:

Bash
python -m venv venv
# Для Windows (PowerShell):
venv\Scripts\activate
# Для Mac/Linux:
source venv/bin/activate
Установка зависимостей:

Bash
pip install -r requirements.txt
pip install faster-whisper
Запуск сервера:

Bash
uvicorn main:app --reload --port 8000
Сервер запустится на http://127.0.0.1:8000. База данных taskflow.db создастся автоматически.

2. Фронтенд (React / Vite)
Откройте новый терминал и перейдите в папку frontend.

Установка и запуск:

Bash
npm install
npm run dev
Приложение откроется по адресу http://localhost:5173.

🔑 Настройка API ключей (.env)
Для корректной работы ИИ создайте файл .env в папке backend:

Фрагмент кода
# Ключ для работы ИИ-ассистента
OPENAI_API_KEY=ваш_ключ_openai

# Настройки базы данных
DATABASE_URL=sqlite:///./taskflow.db
📂 Структура проекта
frontend/: Интерфейс на React.

vite.config.js: Настроен прокси (/api -> localhost:8000) для обхода CORS.

src/api.js: Конфигурация Axios для связи с бэкендом.

backend/: Логика на FastAPI.

routers/ai_agent.py: Логика обработки команд ИИ.

services/transcribe.py: Сервис для распознавания голоса через faster-whisper.

database.py: Настройка SQLAlchemy и SQLite.

⚠️ Решение частых проблем
Ошибка "Fatal error in launcher": Если вы переместили папку проекта, удалите папку venv и создайте её заново (см. раздел "Бэкенд").

База данных пуста: Убедитесь, что открываете файл taskflow.db в программе DB Browser по тому пути, который печатает сервер в консоли при запуске.

ModuleNotFoundError: faster_whisper: Убедитесь, что виртуальное окружение активно ((venv) в терминале) перед установкой библиотек.
